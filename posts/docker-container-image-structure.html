Docker 容器与镜像基本认识与基础命令 (笔记)

发表于2026-04-02|更新于2026-04-02|信息技术

Docker 分为三个部分

1. 客户端 (Client)

用户与 Docker 交互的入口，通常是命令行工具 docker。它将用户的命令（如 docker run、docker pull）发送给 Docker 守护进程，并显示执行结果。

2. 服务端 (Server)

运行 Docker 守护进程和容器的物理机或虚拟机。核心包括：

  • Docker daemon：后台常驻进程，负责管理镜像、容器、网络、卷等资源
  • 镜像 (Images)：只读模板，包含应用程序及运行环境
  • 容器 (Containers)：镜像的运行实例，彼此隔离
  • 网络 (Networks) 和 卷 (Volumes)：管理容器通信和数据持久化

3. 仓库 (Registry)

存储和分发镜像的服务，如 Docker Hub。通过 docker pull 从仓库下载镜像，通过 docker push 上传本地镜像到仓库。

三个核心概念

1. 镜像 (Image)

  • 相当于一个安装包或模板
  • 里面打包了程序 + 运行环境（比如一个带 Python 3.9 和 Flask 的 Ubuntu 系统）
  • 只读，不能直接运行

2. 容器 (Container)

  • 相当于运行起来的程序实例
  • 镜像像一个类，容器就是 new 出来的对象
  • 一个镜像可以启动多个容器，彼此隔离

3. 仓库 (Registry)

  • 相当于应用商店，存放镜像
  • Docker Hub 是官方仓库，可以下载别人做好的镜像（如 MySQL、Nginx）

基础示例

以 nginx 为例，当我们需要在服务器上部署一个已经打包好的应用和运行环境时，只需下载并运行打包好的镜像，即可完成部署。

docker run -d -p 8080:80 --name my-nginx nginx

它做了什么？

  1. docker run
    告诉 Docker 守护进程：请创建一个新容器并运行它。

  2. -d（detach 模式）
    让容器在后台运行。如果不加这个参数，终端会被容器“占用”，关闭终端容器就会停止。

  3. -p 8080:80（端口映射）
    把容器的 80 端口映射到宿主机的 8080 端口。
    localhost:8080(宿主机) --> :80 (Nginx 监听的端口)
    当你在浏览器访问 http://localhost:8080，请求会被转发到容器里的 Nginx。

  4. --name my-nginx
    给容器起个名字叫 my-nginx，方便后续管理（docker stop my-nginx）。

  5. nginx
    指定使用哪个镜像。如果本地没有这个镜像，Docker 会自动从 Docker Hub 下载。

常用命令

命令 作用
docker pull 镜像名 下载镜像
docker images 查看本地镜像
docker run 镜像名 用镜像创建并启动容器
docker ps 查看运行中的容器
docker stop 容器ID 停止容器
docker rm 容器ID 删除容器
docker rmi 镜像ID 删除镜像

容器与镜像

1. docker pull 拉取镜像

docker pull 是用于从镜像仓库（如 Docker Hub）下载镜像到本地的命令。

docker pull [镜像名称]:[标签]

拉取最新版本的 nginx 镜像:

docker pull nginx

拉取指定版本的 nginx:

docker pull nginx:1.25

从指定仓库拉取:

docker pull registry.cn-hangzhou.aliyuncs.com/xxx/nginx

2. docker run 运行镜像

镜像下载之后会存放在本地，要启动这个镜像的容器需要使用 docker run。

docker run [选项] [镜像名称]:[标签] [命令]

最简单的运行方式（前台运行，会占用终端）:

docker run nginx

后台运行:

docker run -d nginx

指定容器名称:

docker run -d -p 8080:80 --name my-nginx nginx

docker run 执行过程:

  1. 检查本地是否存在指定的镜像
  2. 如果不存在，自动从 Docker Hub 拉取（相当于先 pull 再 run）
  3. 基于镜像创建容器
  4. 分配文件系统、网络 IP、挂载点等资源
  5. 启动容器内的进程（如 Nginx 服务）

一个镜像启动多个容器(基于同一个 nginx 镜像，启动三个独立容器):

docker run -d --name web1 -p 8081:80 nginx
docker run -d --name web2 -p 8082:80 nginx
docker run -d --name web3 -p 8083:80 nginx

三个容器彼此隔离，互不影响，访问不同端口都能看到 Nginx 欢迎页。

参数 作用说明
-d 后台运行容器（detach 模式），容器不会占用当前终端
--name 为容器指定名称，便于管理和后续操作
-p 端口映射，格式：宿主机端口:容器端口，用于访问容器服务
-v 挂载卷，将宿主机目录映射到容器内，实现数据持久化或共享
-e 设置环境变量，在容器运行时传递配置参数
--rm 容器停止后自动删除，避免产生无用的容器残留
-it 交互式运行，进入容器内部操作（结合 bash 或 sh 使用）

查看和管理容器
查看所有容器(正在运行的)：

docker ps

查看所有容器（包括已停止的）:

docker ps -a

停止容器:

docker stop my-nginx

启动已停止的容器:

docker start my-nginx

重启容器：

 docker restart my-nginx

删除容器（需先停止）:

docker rm my-nginx

查看容器日志:

docker logs my-nginx

3. docker create 创建容器（不运行）

如果只想创建容器但不启动，可以使用 docker create 命令。

docker create [选项] [镜像名称]:[标签] [命令]

与 docker run 的区别：

命令 作用 状态
docker run 创建 + 启动 容器直接运行
docker create 仅创建 容器处于“已创建”状态，不运行

创建事例：

docker create -p 8080:80 --name my-nginx nginx

查看所有容器（包括已创建的）：

docker ps -a

输出中容器状态为 Created，而不是 Up。

启动已创建的容器：

docker start my-nginx

查看运行状态:

docker ps

此时状态变为 Up。

镜像与容器的关系

镜像: 安装包 / 静态文件，不运行。
容器: 运行中的程序 / 动态进程，占用资源。

镜像下载一次，可以启动无数个容器。容器删除后数据会丢失，如需持久化需使用卷（Volume）。
